# TaxMasterKZ — универсальный помощник камерального анализа налогоплательщика

Приложение для налогового инспектора / консультанта: загружаете разные выгрузки
по налогоплательщику (ЭСФ, ФНО, карточка НП из КГД/SUR и т.д.), приложение
приводит их к единому формату, сопоставляет между собой, выявляет расхождения
и формирует аналитическую справку по НП простым языком.

Приложение универсально: не привязано к конкретному БИН, году или набору
файлов — все парсеры терпимы к разным названиям колонок, разным годам,
неполному набору файлов, «грязным» Excel-выгрузкам (см. раздел «Как это
работает» ниже).

📖 **Пошаговая инструкция для налогового специалиста** (без программистских
терминов) — файл [`ИНСТРУКЦИЯ_ПОЛЬЗОВАТЕЛЯ.md`](./ИНСТРУКЦИЯ_ПОЛЬЗОВАТЕЛЯ.md)
в корне проекта, либо раздел **«Инструкция»** прямо внутри приложения (первый
пункт бокового меню, также доступен с главной страницы по кнопке
«📖 Открыть инструкцию»).

🚀 **Развёртывание как онлайн-сервис** (Docker, VPS, домен, резервные копии,
управление пользователями) — файл [`README_DEPLOY.md`](./README_DEPLOY.md).

## Авторизация

С этой версии приложение требует вход по логину/паролю — оно рассчитано на
одновременную работу многих налоговых специалистов через браузер, а не
только на локальный запуск одним человеком. Каждый пользователь видит
только свои загруженные файлы и свои анализы; администратор управляет
доступом через раздел «Админ-панель» (создание пользователей, блокировка,
срок доступа, режим обслуживания). Подробности — в README_DEPLOY.md,
раздел «Как создать первого администратора».

## Тарифная система (v4)

С версии v4 доступ к сервису продаётся по тарифам. Тариф определяет срок
доступа, лимит анализов в месяц/период, разрешённые форматы экспорта и
наличие водяного знака в справке:

| Тариф | Цена | Срок | Лимит анализов | Пользователей | Экспорт |
|---|---|---|---|---|---|
| **DEMO** | 0 ₸ | 7 дней | 2 | 1 | Word/PDF с водяным знаком, Excel закрыт |
| **START** | 59 000 ₸/мес | 1 месяц | 20 | 1 | Word/PDF, Excel — только основные своды |
| **PRO** | 129 000 ₸/мес | 1 месяц | 100 | 1 | Word/PDF/Excel полностью, расширенный риск-анализ |
| **TEAM** | 250 000 ₸/мес | 1 месяц | 300 | до 5 | Полный экспорт, командный доступ |
| **ENTERPRISE** | индивидуально | задаёт админ | задаёт админ | без ограничения | Полный экспорт, индивидуальные настройки |

Тарифы хранятся в таблице `plans` и редактируются администратором прямо в
приложении (раздел «Админ-панель» → «Редактирование тарифов») — цену, срок,
лимит, разрешённые форматы и описание можно менять без правки кода.

Пользователи, созданные ещё в версии v3 (без тарифа), продолжают работать
без ограничений — лимит анализов, водяной знак и запрет Excel применяются
только к пользователям с назначенным тарифом. Это осознанное решение для
обратной совместимости при обновлении с v3 на v4 (подробности — в
README_DEPLOY.md, раздел «Обновление с v3 на v4»).

Каждый пользователь видит свой тариф, остаток анализов, срок действия и
статус оплаты на странице **«Мой тариф»**; страница **«Тарифы»** — публичная
витрина с карточками тарифов и кнопками связи (WhatsApp/Threads).

Пошаговые инструкции для администратора (как создать пользователя с
тарифом, продлить доступ, сбросить лимит, отметить оплату, заблокировать) —
в README_DEPLOY.md.

## Запуск

```bash
pip install -r requirements.txt
cp .env.example .env
streamlit run app.py
```

При первом запуске автоматически создаётся администратор из `.env`
(логин/пароль по умолчанию `admin` / `changeme123` — смените после первого
входа). Также можно запустить через Docker: `docker compose up -d`
(см. README_DEPLOY.md).

Откроется браузер на `http://localhost:8501`. Никакой дополнительной
настройки (БД, ключей API) не требуется — приложение полностью локальное.

Проверено на Python 3.10+.

## Значок Threads

В боковом меню (видно на каждой странице, под счётчиком загруженных файлов)
и внизу главной страницы есть кликабельный значок **«TaxMasterKZ в Threads»**
— круглый чёрно-белый значок «@» с подписью. Клик открывает
`https://www.threads.com/@taxmasterkz` в новой вкладке браузера.

**Как проверить, что значок работает:** откройте приложение, посмотрите
низ бокового меню (или низ главной страницы) — значок должен быть виден
без прокрутки в обычном окне браузера. Наведите курсор — должна появиться
"рука"/подсветка ссылки; клик открывает Threads в новой вкладке. Значок
реализован как обычная HTML-ссылка `<a href="...">` через
`st.markdown(..., unsafe_allow_html=True)` (см. функцию `threads_badge_html()`
в `app.py`) — работает во всех браузерах без JavaScript.

На странице **«Загрузка данных»** — 10 вкладок:

| Вкладка              | Что грузить                                                              | Формат        |
|-----------------------|---------------------------------------------------------------------------|---------------|
| Карточка НП           | Сведения по НП из КГД/SUR (БИН, ОКЭД, НДС, численность, КНН и т.д.)      | PDF, Excel    |
| ФНО                    | Реестры деклараций 100.00 / 200.00 / 200.01 / 300.00 и др.               | Excel, CSV    |

| ЭСФ реализация         | Выгрузки ЭСФ по продажам (за любой год)                                  | Excel, CSV    |
| Банк / 1С / СНТ / ЭАВР / Контрагенты / Прочее | Дополнительные источники (см. «Что уже умеет» ниже)  | Excel, CSV, PDF |

Файлы можно грузить в любом порядке и не все сразу — приложение работает и
с неполным набором (например, только ЭСФ без ФНО). Там, где данных не
хватает для вывода, приложение честно пишет «Недостаточно данных для
окончательного вывода. Необходимо загрузить: …» вместо того, чтобы гадать.

После загрузки нажмите **«🔄 Собрать базу и пересчитать аналитику»** —
это единая точка пересчёта всех разделов (Дашборд, Покупатели, Поставщики,
Камеральный контроль, Генератор справки).

## Как это работает (коротко про универсальность)

1. **file_loader.py** открывает Excel/CSV и сам находит строку заголовка
   среди первых 15 строк (в тестовых выгрузках КГД заголовок ЭСФ — 3-я
   строка; в других файлах может быть 1-я или 5-я — не важно).
2. **column_mapper.py** сопоставляет разные названия колонок одному
   внутреннему имени через словарь синонимов + нечёткое совпадение
   (`difflib`), например «БИН», «ИИН/БИН», «БИН(ИИН)» → `bin_iin`.
3. **data_normalizer.py** приводит к единому виду БИН (в т.ч. если Excel
   превратил его в `5.1240011241e+11` или обрезал ведущий ноль), даты
   (`dd.mm.yyyy`, `yyyy-mm-dd`, Excel serial number) и суммы (текст,
   пробелы, запятые).
4. **esf_parser.py** отдельно решает специфичную для ЭСФ-выгрузок КГД
   задачу: колонка «ИИН/БИН» встречается дважды с одинаковым названием
   (для поставщика и покупателя) — направление определяется по позиции
   и, если известен БИН налогоплательщика (из карточки НП), сверяется
   автоматически с фактическим направлением сделки.
5. Дубли строк ЭСФ, дополнительные/исправленные ЭСФ и «поздние» ЭСФ не
   удаляются — помечаются и попадают в риск-движок как отдельные находки.

## Важно про ФНО и суммовые показатели деклараций

Реальные выгрузки реестров ФНО из кабинета налогоплательщика (проверено на
тестовых файлах 100.00 / 200.00 / 200.01 / 300.00) содержат не «человеческие»
названия строк декларации, а внутренние технические коды полей ИС СОНО вида
`field_300_00_024_02`. Официальной публичной расшифровки «код поля → строка
декларации» КГД не публикует.

Поэтому приложение:
- **всегда** надёжно распознаёт метаданные декларации (БИН, вид — очередная/
  дополнительная, регистрационный номер, статус, даты подачи/приёма, период) —
  это видно на странице «ФНО» сразу после загрузки;
- **не гадает** суммы (доход, НДС начисленный, НДС в зачёт, ИПН, ОПВ и т.д.) —
  пока вы не сопоставите нужные коды полей в `config/fno_field_map.json`
  (страница **«Настройки»** в приложении — там же можно отредактировать файл
  прямо из интерфейса).

Пример записи в `fno_field_map.json`:

```json
{
  "300.00": {
    "field_300_00_004": {"label": "Оборот по реализации, всего", "role": "vat_charged"}
  }
}
```

Доступные роли: `income`, `cit_amount`, `pit_amount`, `vat_charged`,
`vat_credit`, `vat_payable`, `payroll_ipn`, `social_tax`, `opv`, `so`,
`vosms`, `opvr`. После сохранения карты — заново нажмите «Собрать базу и
пересчитать аналитику»: сверка ЭСФ vs ФНО 300.00/100.00 сразу заработает
с реальными суммами вместо пометки «нет данных».

**Это осознанное архитектурное решение**, а не недоработка: показывать
непроверенную сумму под видом «НДС по декларации» в справке, которая пойдёт
в акт проверки или клиенту, — риск хуже, чем честно написать «нет данных».

## Логика риск-движка (modules/risk_engine.py)

Все находки формируются по единому шаблону `_finding()`, который жёстко
разделяет факт/расчёт/риск/рекомендацию и **никогда** не делает
категоричных обвинительных выводов (см. п.9 ТЗ). Запрещённые формулировки
("уклонялся от налогов", "сделки фиктивные", "обналичивание доказано") в
кодовой базе не используются нигде — только "и
