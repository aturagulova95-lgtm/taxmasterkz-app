# TaxMasterKZ — инструкция по развёртыванию

Простое руководство для запуска локально, на VPS и через Docker. Рассчитано
на человека без глубокого опыта администрирования серверов.

---

## 1. Как запустить локально (без Docker)

```bash
pip install -r requirements.txt
cp .env.example .env
streamlit run app.py
```

При **первом** запуске приложение само создаст администратора с логином и
паролем из `.env` (`ADMIN_USERNAME` / `ADMIN_PASSWORD`, по умолчанию
`admin` / `changeme123`). Откройте `http://localhost:8501`, войдите под
этими данными и **сразу смените пароль** в Админ-панели.

---

## 2. Как запустить на VPS (без Docker)

1. Арендуйте VPS (например, 2 ГБ ОЗУ достаточно для MVP) с Ubuntu 22.04.
2. Установите Python 3.11+ и git:
   ```bash
   sudo apt update && sudo apt install -y python3 python3-pip python3-venv
   ```
3. Скопируйте проект на сервер (scp/git clone) в `/opt/taxmasterkz`.
4. Создайте виртуальное окружение и установите зависимости:
   ```bash
   cd /opt/taxmasterkz
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   cp .env.example .env && nano .env   # впишите свой ADMIN_PASSWORD
   ```
5. Запустите приложение на всех интерфейсах, в фоне (через `systemd`,
   рекомендуется) или через `screen`/`tmux` для быстрой проверки:
   ```bash
   streamlit run app.py --server.address=0.0.0.0 --server.port=8501
   ```
6. Откройте `http://<IP-адрес-сервера>:8501` в браузере.

**Рекомендуется** для боевой эксплуатации использовать Docker (раздел 3) —
он проще в обслуживании и обновлении.

### Автозапуск через systemd (без Docker)

Создайте `/etc/systemd/system/taxmasterkz.service`:

```ini
[Unit]
Description=TaxMasterKZ
After=network.target

[Service]
WorkingDirectory=/opt/taxmasterkz
ExecStart=/opt/taxmasterkz/venv/bin/streamlit run app.py --server.address=0.0.0.0 --server.port=8501 --server.headless=true
Restart=always
EnvironmentFile=/opt/taxmasterkz/.env

[Install]
WantedBy=multi-user.target
```

Затем:
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now taxmasterkz
```

## 3. Как запустить через Docker (рекомендуется)

На VPS с установленным Docker и Docker Compose:

```bash
git clone <ваш-репозиторий> taxmasterkz && cd taxmasterkz
cp .env.example .env
nano .env   # впишите ADMIN_USERNAME / ADMIN_PASSWORD / THREADS_URL
docker compose up -d
```

Приложение поднимется на порту 8501 (`http://<IP-сервера>:8501`).
Все данные (пользователи, файлы, отчёты) сохраняются в папке `./data` на
хосте — при обновлении образа (`docker compose up -d --build`) они не
теряются.

Проверить, что контейнер запущен:
```bash
docker compose ps
docker compose logs -f
```

Остановить:
```bash
docker compose down
```

## 4. Как создать первого администратора

Первый администратор создаётся **автоматически** при первом запуске
приложения (локально, на VPS или в Docker) — из переменных `ADMIN_USERNAME`
и `ADMIN_PASSWORD` в файле `.env`. Если `.env` не создан — используются
значения по умолчанию `admin` / `changeme123` (обязательно смените пароль
после первого входа!).

Если нужно создать администратора вручную (например, файл `.env` не был
готов при первом запуске и уже создан пользователь с другой ролью) —
выполните из папки проекта:

```bash
python -c "
from modules import auth_db
auth_db.init_db()
ok, msg = auth_db.create_user('admin2', 'НадёжныйПароль123', role='admin')
print(ok, msg)
"
```

(В Docker: `docker compose exec taxmasterkz python -c "..."` с той же командой.)

## 5. Как открыть доступ пользователям (создать нового пользователя)

1. Войдите под администратором.
2. Откройте раздел **«Админ-панель»** в боковом меню → блок **«Тарифы и
   доступ»** → разверните **«➕ Создать пользователя»**.
3. Укажите логин, пароль, роль (`admin` / `user` / `demo`) и **тариф**
   (DEMO / START / PRO / TEAM / ENTERPRISE или «— без тарифа —»).
4. При выборе тарифа приложение автоматически проставит дату начала/окончания
   доступа, лимит анализов и статус оплаты по умолчанию для этого тарифа —
   вручную ничего дополнительно указывать не нужно (детали — в разделе 11
   ниже).
5. При желании заполните компанию/телефон/email/комментарий.
6. Нажмите «Создать пользователя».
7. Передайте логин и пароль пользователю удобным вам способом (лично,
   защищённым каналом — не по открытой почте).

## 6. Как заблокировать пользователя

В Админ-панели найдите нужного пользователя в списке, разверните карточку
и нажмите **«Заблокировать»**. Доступ блокируется немедленно — при
следующем действии в приложении (даже если пользователь уже был
авторизован) он увидит сообщение о блокировке и будет разлогинен.
Разблокировка — той же кнопкой (после блокировки она превращается в
«Разблокировать»).

Чтобы удалить пользователя насовсем — кнопка **«🗑 Удалить»** в той же
карточке (нельзя удалить самого себя, во избежание потери доступа к
Админ-панели).

---

## 7. Как включить режим обслуживания

В Админ-панели, в самом верху — переключатель **«Режим обслуживания»**.
При включении все пользователи, кроме администраторов, при заходе в
приложение видят сообщение:

> Сервис временно недоступен. По вопросам доступа обратитесь в TaxMasterKZ.

Администраторы продолжают работать в обычном режиме — это нужно, чтобы
можно было провести обслуживание/обновление и затем выключить режим той
же кнопкой.

## 8. Как заменить ссылки Threads и WhatsApp

Ссылки задаются переменными окружения `THREADS_URL` и `WHATSAPP_URL` в файле `.env`:

```
THREADS_URL=https://www.threads.com/@ваш_точный_профиль
WHATSAPP_URL=https://wa.me/77001234567
```

(Необязательно) контактный email поддержки:

```
SUPPORT_EMAIL=support@taxmasterkz.kz
```

После изменения `.env`:
- локально/VPS без Docker — перезапустите `streamlit run app.py`;
- в Docker — выполните `docker compose up -d` (пересоздаст контейнер с
  новыми переменными окружения).

Если переменные не заданы — используются базовые значения: `THREADS_URL`
= `https://www.threads.com/`, `WHATSAPP_URL` = `https://wa.me/77000000000`,
`SUPPORT_EMAIL` — пусто (значок email просто не показывается). Ссылки видны
на страницах «Тарифы», «Мой тариф», «Доступ», на экране входа и на экранах
блокировки/истечения доступа/режима обслуживания.

## 9. Как подключить домен

1. Настройте A-запись домена на IP-адрес вашего сервера.
2. Установите reverse-proxy с TLS — проще всего через **Caddy** (сам
   получает и обновляет сертификат Let's Encrypt) или **nginx + certbot**.

Пример `Caddyfile` (если используете Caddy):
```
tax.вашдомен.kz {
    reverse_proxy localhost:8501
}
```
Запуск Caddy: `caddy run` (или как systemd-сервис/Docker-контейнер).

Пример конфигурации nginx (если используете nginx + certbot):
```nginx
server {
    listen 80;
    server_name tax.вашдомен.kz;
    location / {
        proxy_pass http://localhost:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```
Затем: `sudo certbot --nginx -d tax.вашдомен.kz` для HTTPS.

## 10. Как сделать резервную копию

Все данные приложения (пользователи, пароли-хэши, журналы, загруженные
файлы, результаты анализа) лежат в одной папке — `./data`. Резервная копия
сводится к копированию этой папки:

```bash
tar -czf taxmasterkz_backup_$(date +%Y%m%d).tar.gz data/
```

- делать это регулярно по расписанию (`cron`), например раз в сутки;
