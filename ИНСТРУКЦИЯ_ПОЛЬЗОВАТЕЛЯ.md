# Инструкция пользователя — TaxMasterKZ

Простое руководство для налогового специалиста. Программистские знания не нужны.

---

## 0. Вход в систему

Приложение — общий онлайн-сервис: логин и пароль вам выдаёт администратор
TaxMasterKZ (сами вы зарегистрироваться не можете). Откройте ссылку сервиса,
введите логин и пароль на экране входа и нажмите «Войти».

Если пишет «Ваш доступ заблокирован», «Ваш доступ завершён» или «Лимит
анализов по вашему тарифу исчерпан» — обратитесь к администратору (кнопки
WhatsApp и Threads показаны прямо на этом экране, либо см. раздел «Доступ»
внутри приложения). Все ваши загруженные файлы и анализы видны только
вам — другие пользователи сервиса их не видят.

---

## 0.1. Тарифы и лимит анализов

Доступ к сервису предоставляется по тарифу (DEMO / START / PRO / TEAM /
ENTERPRISE) — тариф определяет срок действия доступа, сколько анализов
можно запустить, и в каком формате можно скачать справку.

- Ваш текущий тариф, срок действия, сколько анализов осталось и статус
  оплаты — на странице **«Мой тариф»** (пункт бокового меню).
- Сравнить тарифы и узнать, как перейти на более широкий — страница
  **«Тарифы»**.
- **Что считается «анализом»**: только нажатие кнопки «🔄 Собрать базу и
  пересчитать аналитику» на странице «Загрузка данных». Простой просмотр
  страниц, повторное открытие уже готового анализа и скачивание ранее
  сформированного отчёта лимит **не расходуют**.
- Если лимит анализов исчерпан — кнопка пересчёта становится неактивной, и
  показывается сообщение с предложением перейти на более широкий тариф или
  обратиться в TaxMasterKZ (WhatsApp/Threads — там же).
- На тарифе **DEMO** экспорт в Excel закрыт, а Word/PDF-справка формируется
  с водяным знаком «Демо-версия TaxMasterKZ. Для полного доступа обратитесь
  в TaxMasterKZ.» — это не ошибка, а особенность демо-доступа.
- Если до конца срока доступа остаётся 3 дня или меньше, на странице «Мой
  тариф» появляется предупреждение о скором завершении доступа — заранее
  свяжитесь с администратором, чтобы не потерять доступ к своим анализам.

## 1. Для чего нужно приложение

**TaxMasterKZ** помогает налоговому специалисту быстро провести предварительный
камеральный анализ налогоплательщика (НП):

- загрузить данные по НП;
- собрать ЭСФ по реализации и приобретению;
- загрузить ФНО (налоговые декларации);
- сформировать список покупателей и поставщиков по годам;
- сопоставить ЭСФ с декларациями;
- выявить расхождения;
- определить налоговые риски;
- подготовить аналитическую справку;
- сформировать список документов, которые нужно запросить у налогоплательщика.

Всё это — за минуты вместо ручного сведения таблиц в Excel.

---

## 2. Какие файлы можно загружать

### 1. Карточка налогоплательщика
- сведения из КГД/SUR;
- формат: PDF или Excel;
- данные: БИН, ОКЭД, НДС-статус, численность, налоговая нагрузка, задолженность,
  признаки неблагонадёжности.

### 2. ЭСФ по реализации
- Excel по годам (можно за один год, можно за несколько — загружайте отдельными файлами);
- название файла не важно: «реализация 2023», «ЭСФ реализация», «sales» — приложение
  само разберётся.

### 3. ЭСФ по приобретению
- Excel по годам;
- название файла не важно: «приобретение 2023», «ЭСФ покупки», «purchases».

### 4. ФНО (налоговая отчётность)
- 100.00 (декларация по КПН);
- 200.00 / 200.01 (зарплатные налоги);
- 300.00 (декларация по НДС);
- в будущих версиях — 910.00, 220.00, 328.00, 701.00, 870.00 и другие формы.

### 5. Дополнительные файлы
- банковские выписки;
- оборотно-сальдовые ведомости (ОСВ) из 1С;
- реестры СНТ;
- реестры ЭАВР;
- сведения по контрагентам.

> Эти файлы сейчас загружаются и показываются в предпросмотре (сколько строк,
> какой файл), но глубокий разбор по ним — в следующей версии приложения.

**Важно:** не обязательно грузить всё сразу. Приложение работает и с неполным
набором файлов — просто по разделам, для которых данных не хватает, оно честно
напишет «Недостаточно данных для окончательного вывода».

## 3. Как запустить приложение

1. Установите Python (версия 3.10 или новее) — [python.org](https://www.python.org/downloads/).
2. Откройте папку проекта (там, где лежит файл `app.py`).
3. Установите зависимости командой:

   ```bash
   pip install -r requirements.txt
   ```

4. Запустите приложение:

   ```bash
   streamlit run app.py
   ```

5. После запуска автоматически откроется браузер с приложением
   (если не открылся — перейдите по адресу `http://localhost:8501`).

## 4. Как работать с приложением пошагово

**Шаг 1.** Открыть приложение (`streamlit run app.py`).

**Шаг 2.** Перейти в раздел «Загрузка данных» (можно нажать кнопку «🚀 Начать
анализ» на главной странице).

**Шаг 3.** Загрузить карточку НП (вкладка «Карточка НП»).

**Шаг 4.** Загрузить ЭСФ по реализации по годам (вкладка «ЭСФ реализация»).

**Шаг 5.** Загрузить ЭСФ по приобретению по годам (вкладка «ЭСФ приобретение»).

**Шаг 6.** Загрузить ФНО 100.00, 200.00, 200.01, 300.00 (вкладка «ФНО»).

**Шаг 7.** Проверить в таблице предпросмотра внизу страницы, правильно ли
приложение определило:
- реализация или приобретение;
- какая форма ФНО;
- карточка НП;
- год и период.

Если что-то определено неверно — см. раздел 7 «Что делать, если файл не
распознался».

**Шаг 8.** Нажать кнопку **«🔄 Собрать базу и пересчитать аналитику»**.

**Шаг 9.** Перейти в «Дашборд» и посмотреть общие итоги (суммы реализации,
приобретений, НДС, количество контрагентов).

**Шаг 10.** Перейти в «Покупатели» и проверить список покупателей по годам
(с фильтрами по году, сумме, поиском по названию/БИН).

**Шаг 11.** Перейти в «Поставщики» и проверить список поставщиков по годам
(там же — группировки: с НДС / без НДС, с высокой концентрацией, разовые).

**Шаг 12.** Перейти в «Камеральный контроль» и посмотреть сверку ЭСФ и ФНО
(расхождения по НДС и по доходу).

**Шаг 13.** Там же, ниже — раздел «Найденные риски»: посмотреть выявленные
зоны риска, при необходимости скорректировать уровень риска или отметить,
что документы по находке уже получены.

**Шаг 14.** Перейти в «Генератор справки».

**Шаг 15.** Выбрать период (годы) и стиль справки:
- для налогового инспектора — подробно, со всеми таблицами;
- для клиента — простым языком, без лишней терминологии;
- для руководителя — коротко, только выводы и суммы;
- для суда/возражения — только факты и документы.

**Шаг 16.** Нажать «📝 Сформировать справку» — справка появится на экране,
ниже будут кнопки скачивания Word, HTML и Excel-приложений к справке.

## 5. Как читать результаты

- **«Реализация»** — это данные по выписанным ЭСФ (что НП продал).
- **«Приобретение»** — это данные по входящим ЭСФ (что НП купил).
- **«Покупатели»** — кому НП реализовывал товары/работы/услуги.
- **«Поставщики»** — у кого НП приобретал товары/работы/услуги.
- **«НДС начисленный»** — НДС по реализации.
- **«НДС в зачёт»** — НДС по приобретениям.
- **«Расхождение»** — разница между данными ЭСФ и данными ФНО за тот же период.
- **«Риск»** — это **не доказанное нарушение**, а участок, который нужно
  проверить документально.

> **Важно.** Приложение не заменяет налоговую проверку и не делает окончательные
> юридические выводы. Оно помогает быстро найти несоответствия и подготовить
> предварительную аналитическую справку. Все окончательные выводы делаются
> только после изучения первичных документов.

## 6. Что делать, если приложение показывает расхождение

1. Проверить период (год/квартал), к которому относится расхождение.
2. Проверить, все ли нужные файлы загружены (и ЭСФ, и соответствующая ФНО).
3. Проверить, нет ли дополнительных или исправленных ЭСФ за этот период.
4. Проверить статус ЭСФ (не отозвана/аннулирована ли она).
5. Проверить дату оборота и дату выписки ЭСФ — не выписана ли она значительно
   позже даты оборота (это может сдвигать период отражения).
6. Проверить, в каком периоде НДС фактически отнесён в зачёт.
7. Проверить соответствующую строку ФНО (сверить вручную с декларацией
   в кабинете налогоплательщика).
8. При необходимости — запросить документы у НП (см. раздел 8).

## 7. Что делать, если файл не распознался

Если приложение не распознало файл:

- проверьте, что файл в формате Excel (.xlsx/.xls), CSV или PDF;
- проверьте, что в файле есть заголовки колонок (пустые файлы или файлы
  без шапки таблицы приложение не сможет разобрать);
- если заголовки начинаются не с первой строки — не страшно: приложение
  само пытается найти строку заголовка среди первых 15 строк листа;
- проверьте, что файл не защищён паролем — с запароленными файлами
  приложение работать не может;
- попробуйте пересохранить файл в Excel (иногда помогает, если файл
  скачан в нестандартном формате);
- если после загрузки приложение определило тип файла неверно
  (например, перепутало приобретение и реализацию) — загрузите его через
  правильную вкладку ещё раз: направление задаётся вкладкой, на которую
  вы загружаете файл («ЭСФ приобретение» / «ЭСФ реализация»);
- смотрите колонку «Предупреждения» в таблице предпросмотра — там
  приложение объясняет, что именно не удалось распознать.

## 8. Какие документы запрашивать у НП

### По реализации
- договоры;
- акты;
- ЭАВР;
- накладные;
- ЭСФ;
- СНТ;
- платёжные документы.

### По приобретению
- договоры с поставщиками;
- счета;
- акты;
- ЭАВР;
- накладные;
- ЭСФ;
- СНТ;
- платёжные поручения;
- транспортные документы;
- подтверждение фактического получения товаров/работ/услуг.

### По зарплате
- штатное расписание;
- трудовые договоры;
- табели учёта рабочего времени;
- расчётные ведомости;
- платёжные документы по зарплате;
- договоры ГПХ (гражданско-правового характера).

Раздел «Камеральный контроль» → «Найденные риски» формирует такой список
автоматически для каждой конкретной находки — привязанный к конкретному
контрагенту и периоду.

## 9. Частые ошибки пользователей

- загрузили только реализацию, но не загрузили приобретение (или наоборот);
- загрузили не все годы — часть периодов выпала из анализа;
- перепутали приобретение и реализацию (загрузили не на ту вкладку);
- з
