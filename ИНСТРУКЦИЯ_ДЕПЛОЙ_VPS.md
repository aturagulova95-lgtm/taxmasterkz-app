# TaxMasterKZ — деплой на VPS (production), пошагово для новичка

Эта инструкция написана для человека без опыта работы с Linux-серверами.
Каждый шаг — с конкретными командами (их нужно скопировать и вставить в
терминал) и пояснением, что происходит и как проверить результат.

Пример домена в инструкции: `analysis.taxmaster.kz` — замените на свой
везде, где он встречается.

---

## 1. Как купить/подготовить VPS Ubuntu 22.04/24.04

VPS (виртуальный сервер) можно арендовать у любого провайдера — например,
Timeweb Cloud, Selectel, Beget, DigitalOcean, Hetzner и т.д. Точные названия
кнопок отличаются, но общий порядок везде одинаковый:

1. Зарегистрируйтесь на сайте провайдера, привяжите способ оплаты.
2. Закажите новый сервер (обычно кнопка «Создать сервер» / «Order VPS» /
   «Deploy new server»).
3. При выборе образа операционной системы выберите **Ubuntu 22.04 LTS**
   или **Ubuntu 24.04 LTS** (LTS — long term support, самые стабильные версии).
4. Минимальная конфигурация для TaxMasterKZ: **2 ГБ ОЗУ, 2 ядра CPU, 20+ ГБ
   диска** — этого достаточно для десятков одновременных пользователей.
5. При создании сервера провайдер попросит либо задать пароль root,
   либо загрузить публичный SSH-ключ. Если не знаете, что такое SSH-ключ —
   выберите вариант с паролем, он проще для начала.
6. После создания сервера у вас будет **IP-адрес** (набор из 4 чисел через
   точку, например `185.12.34.56`) — он понадобится для всех следующих шагов.
   Сохраните его и пароль root в надёжном месте.

## 2. Как подключиться к серверу по SSH

SSH — это защищённый способ управлять сервером через терминал со своего
компьютера.

**На Windows:** откройте приложение «Терминал» (или PowerShell — найдите
через поиск в Пуске). На Mac/Linux: приложение «Терминал».

Введите (замените `185.12.34.56` на IP вашего сервера):

```bash
ssh root@185.12.34.56
```

Если это первое подключение, терминал спросит «Are you sure you want to
continue connecting?» — введите `yes` и нажмите Enter.

Затем введите пароль root, который вы получили при создании сервера
(при вводе пароля символы не отображаются на экране — это нормально,
просто вводите и нажимайте Enter).

Если всё получилось — приглашение в терминале изменится на что-то вроде
`root@имя-сервера:~#` — вы внутри сервера.

## 3. Как установить Docker и Docker Compose

Скопируйте и выполните команды по очереди (каждую — Enter, дождитесь
завершения перед следующей):

```bash
sudo apt update
sudo apt install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
```

```bash
sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/docker.asc
EOF
```

```bash
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

Проверка, что всё установилось:

```bash
docker --version
docker compose version
```

Оба должны вывести номер версии (например, `Docker version 27.x.x`) без ошибок.

*(Необязательно, для удобства)* чтобы не писать `sudo` перед каждой командой
docker:

```bash
sudo usermod -aG docker $USER
newgrp docker
```

## 4. Как загрузить taxmasterkz_v3.zip на сервер

Есть два способа.

**Способ А — через SFTP-программу (проще для новичка).** Установите на свой
компьютер [FileZilla](https://filezilla-project.org/) или используйте
проводник Windows (если поддерживает SFTP). Подключитесь к серверу:
- Хост: `sftp://185.12.34.56` (ваш IP)
- Логин: `root`
- Пароль: пароль root
- Порт: `22`

Перетащите файл `taxmasterkz_v3.zip` в домашнюю папку сервера (`/root/`).

**Способ Б — командой scp с вашего компьютера** (выполняется НЕ на сервере,
а в терминале на вашем компьютере, из папки, где лежит zip-файл):

```bash
scp taxmasterkz_v3.zip root@185.12.34.56:/root/
```

Введите пароль root, когда попросит. После загрузки файл будет лежать в
`/root/taxmasterkz_v3.zip` на сервере.

## 5. Как распаковать приложение в /opt/taxmasterkz

Снова в SSH-сессии на сервере:

```bash
sudo apt install -y unzip
sudo mkdir -p /opt/taxmasterkz
sudo unzip /root/taxmasterkz_v3.zip -d /opt/taxmasterkz_tmp
sudo mv /opt/taxmasterkz_tmp/taxapp/* /opt/taxmasterkz/
sudo rm -rf /opt/taxmasterkz_tmp
cd /opt/taxmasterkz
ls
```

Команда `ls` должна показать список файлов проекта: `app.py`, `Dockerfile`,
`docker-compose.yml`, `modules/` и т.д. Если видите их — распаковка прошла
успешно.

## 6. Как создать .env с ADMIN_USERNAME, ADMIN_PASSWORD, THREADS_URL

```bash
cd /opt/taxmasterkz
cp .env.example .env
nano .env
```

Откроется текстовый редактор `nano` прямо в терминале. Отредактируйте
строки (стрелками перемещайтесь, печатайте как обычно):

```
ADMIN_USERNAME=admin
ADMIN_PASSWORD=ВашНадёжныйПарольЗдесь
THREADS_URL=https://www.threads.com/@taxmasterkz
```

Используйте длинный, сложный пароль (не менее 12 символов, буквы+цифры).

Когда закончите редактировать: нажмите `Ctrl+O` (сохранить), затем `Enter`,
затем `Ctrl+X` (выйти из редактора).

Проверить содержимое файла:
```bash
cat .env
```

## 7. Как запустить docker compose up -d

```bash
cd /opt/taxmasterkz
docker compose up -d
```

Флаг `-d` означает «в фоне» (detached) — терминал не будет занят, приложение
продолжит работать после того, как вы закроете SSH-сессию.

При первом запуске Docker скачает базовый образ Python и соберёт приложение —
это может занять 2–5 минут. Дождитесь, пока команда завершится и вернёт вас
к обычному приглашению терминала.

## 8. Как проверить docker compose logs

```bash
docker compose logs -f
```

Вы увидите поток логов запуска приложения. Ищите строку вида:
```
You can now view your Streamlit app in your browser.
```
Это означает, что приложение запустилось успешно. Чтобы выйти из просмотра
логов (это не останавливает приложение) — нажмите `Ctrl+C`.

Если видите ошибки (`Error`, `Traceback`) — скопируйте текст ошибки, это
поможет быстро найти причину.

Проверить, что контейнер вообще запущен и не падает:
```bash
docker compose ps
```
В колонке `STATUS` должно быть что-то вроде `Up 2 minutes (healthy)`.

## 9. Как открыть приложение по IP:8501

Сначала откройте порт 8501 в файрволе сервера (если файрвол включён):

```bash
sudo ufw allow 8501/tcp
sudo ufw allow OpenSSH
sudo ufw enable
```

(На вопрос про подтверждение — введите `y` и Enter. Если `ufw` уже был
включён, команда `enable` просто подтвердит это.)

Теперь откройте в браузере на своём компьютере:
```
http://185.12.34.56:8501
```
(замените на IP вашего сервера). Должен открыться экран входа TaxMasterKZ.

> ⚠️ Это временный способ для проверки. После настройки домена и HTTPS
> (шаги 10–14) рекомендуется закрыть порт 8501 наружу и оставить доступ
> только через домен — см. примечание в конце шага 14.

## 10. Как подключить домен analysis.taxmaster.kz

Домен должен указывать на IP-адрес вашего сервера через DNS-запись типа A.

Если ваш домен `taxmaster.kz` управляется через **Cloudflare** (рекомендуется —
это понадобится и для шага 20):

1. Зайдите на [dash.cloudflare.com](https://dash.cloudflare.com), выберите
   зону `taxmaster.kz`.
2. Раздел **DNS → Records** → **Add record**.
3. Тип: `A`, Имя: `analysis`, IPv4-адрес: IP вашего сервера (`185.12.34.56`),
   Proxy status: можно оставить **DNS only** (серое облако) на время
   настройки HTTPS через Caddy — так Caddy сможет напрямую получить
   сертификат Let's Encrypt. Включить оранжевое облако (Proxied) можно
   позже, после того как HTTPS уже работает напрямую.
4. Сохраните (**Save**).

Если домен управляется у другого регистратора/DNS-провайдера — принцип тот
же: добавьте A-запись `analysis` → IP сервера в панели управления DNS.

DNS может обновляться от нескольких минут до пары часов. Проверить, что
запись уже видна:
```bash
ping analysis.taxmaster.kz
```
Если в ответе виден IP вашего сервера — можно двигаться дальше.

## 11–13. Как настроить Caddy reverse proxy, HTTPS и Caddyfile

**Установка Caddy** (официальный репозиторий):

```bash
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update
sudo apt install -y caddy
```

Откройте порты 80 и 443 (нужны для HTTP и HTTPS/сертификатов):
```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

**Создание Caddyfile** — это конфигурационный файл, который говорит Caddy,
куда направлять запросы с вашего домена:

```bash
sudo nano /etc/caddy/Caddyfile
```

Удалите всё содержимое (если есть) и вставьте:

```
analysis.taxmaster.kz {
    reverse_proxy localhost:8501
}
```

Сохраните: `Ctrl+O`, `Enter`, `Ctrl+X`.

**Перезапуск Caddy** (шаг 13), чтобы применить конфигурацию:

```bash
sudo systemctl restart caddy
```

Проверить, что Caddy запущен без ошибок:
```bash
sudo systemctl status caddy
```
Должно быть `active (running)` зелёным. Выйти из просмотра статуса — `q`.

Caddy автоматически получит бесплатный HTTPS-сертификат Let's Encrypt при
первом обращении к домену — никаких дополнительных действий не требуется,
если DNS уже настроен правильно (шаг 10).

## 14. Как проверить https://analysis.taxmaster.kz

Откройте в браузере:
```
https://analysis.taxmaster.kz
```
Должен открыться экран входа TaxMasterKZ, а в адресной строке браузера —
значок замочка (защищённое соединение, сертификат получен автоматически).

Если сайт не открывается:
- проверьте `sudo systemctl status caddy` (нет ли ошибок);
- проверьте `docker compose ps` на сервере (контейнер должен быть `Up`);
- проверьте, что DNS-запись действительно указывает на IP сервера (`ping analysis.taxmaster.kz`);
- подождите ещё немного — выпуск сертификата иногда занимает пару минут.

**Рекомендация по безопасности:** после того как домен с HTTPS заработал,
закройте прямой доступ к порту 8501 из интернета (оставив его только для
локального обращения Caddy на самом сервере):
```bash
sudo ufw delete allow 8501/tcp
```
Теперь приложение доступно только через `https://analysis.taxmaster.kz`.

## 15. Как сделать резервное копирование папки data

Все данные приложения (пользователи, пароли-хэши, файлы, отчёты) — в
`/opt/taxmasterkz/data`. Резервная копия:

```bash
cd /opt/taxmasterkz
sudo tar -czf /root/taxmasterkz_backup_$(date +%Y%m%d_%H%M).tar.gz data/
```

Скачайте архив к себе на компьютер (выполняется НЕ на сервере, а в
терминале на вашем компьютере):
```bash
scp root@185.12.34.56:/root/taxmasterkz_backup_*.tar.gz .
```

**Рекомендуется** настроить это по расписанию через `cron`, чтобы не
забывать делать бэкапы вручную:
```bash
sudo crontab -e
```
(при первом запуске спросит редактор — выберите `nano`, вариант 1). Добавьте
строку в конец файла (бэкап каждый день в 03:00):
```
0 3 * * * cd /opt/taxmasterkz && tar -czf /root/backups/taxmasterkz_$(date +\%Y\%m\%d).tar.gz data/
```
Создайте папку для бэкапов заранее: `sudo mkdir -p /root/backups`.
Сохраните файл (`Ctrl+O`, `Enter`, `Ctrl+X`).

## 16. Как обновлять приложение новой версией

1. Сначала сделайте резервную копию (шаг 15) — обязательно перед обновлением!
2. Загрузите новый zip-архив на сервер (шаг 4), например `taxmasterkz_v4.zip`.
3. Остановите текущее приложение:
   ```bash
   cd /opt/taxmasterkz
   docker compose down
   ```
4. Распакуйте новую версию во ВРЕМЕННУЮ папку и скопируйте файлы кода,
   **не трогая папку `data/` и файл `.env`**:
   ```bash
   sudo unzip /root/taxmasterkz_v4.zip -d /opt/taxmasterkz_new
   sudo rsync -a --exclude 'data/' --exclude '.env' /opt/taxmasterkz_new/taxapp/ /opt/taxmasterkz/
   sudo rm -rf /opt/taxmasterkz_new
   ```
   (если `rsync` не установлен: `sudo apt install -y rsync`)
5. Пересоберите и запустите:
   ```bash
   cd /opt/taxmasterkz
   docker compose up -d --build
   ```
6. Проверьте логи (шаг 8) и откройте сайт (шаг 14) — убедитесь, что всё
   работает и данные (пользователи, файлы) на месте.

## 17. Как быстро отключить доступ всем пользователям

Самый быстрый способ — **режим обслуживания** внутри самого приложения
(не требует SSH):
1. Войдите под администратором на `https://analysis.taxmaster.kz`.
2. Раздел **«Админ-панель»** → кнопка **«Включить»** режим обслуживания.

Все обычные пользователи сразу увидят: «Сервис временно недоступен. По
вопросам доступа обратитесь в TaxMasterKZ» — администраторы продолжают
работать.

**Экстренный вариант** (если нет доступа в саму админ-панель) — полностью
остановить приложение через SSH:
```bash
cd /opt/taxmasterkz
docker compose stop
```
Сайт станет недоступен для всех, включая админов. Запустить обратно:
```bash
docker compose start
```

## 18. Как быстро заблокировать одного пользователя

Через SSH это не нужно — всё делается в самом приложении:
1. Войдите под администратором.
2. **«Админ-панель»** → найдите пользователя в списке → разверните карточку →
   кнопка **«Заблокировать»**.

Блокировка применяется мгновенно — при следующем действии в приложении
пользователь будет разлогинен с сообщением о блокировке.

## 20. Как защитить приложение через Cloudflare Access

Это дополнительный уровень защиты: даже до экрана входа TaxMasterKZ
пользователь сначала должен подтвердить свой email через Cloudflare —
полезно, если хотите ограничить доступ только определённым списком людей
или доменом почты (например, только `@taxmaster.kz`), помимо обычного
логина/пароля приложения.

**Требование:** домен `taxmaster.kz` должен управляться через Cloudflare
(шаг 10, вариант с Cloudflare).

1. Зайдите на [dash.cloudflare.com](https://dash.cloudflare.com).
2. В левом меню откройте раздел **Zero Trust**. Если заходите первый раз —
   Cloudflare попросит придумать «team name» (название команды) — можно
   ввести любое, например `taxmasterkz`.
3. В Zero Trust перейдите: **Access controls → Applications**.
4. Нажмите **Add an application** (или **Create application**).
5. Выберите тип **Self-hosted**.
6. В поле **Domain** укажите `analysis.taxmaster.kz` (тот же домен, что
   настраивали в шаге 10).
7. Ниже, в разделе **Policies**, нажмите **Add a policy** (или используйте
   предложенную по умолчанию):
   - **Policy name** — любое понятное имя, например «Только сотрудники».
   - **Action**: `Allow`.
   - В **Include** выберите правило **Emails** и перечислите конкретные
     email-адреса сотрудников, которым разрешён доступ — либо выберите
     **Emails ending in** и укажите `@taxmaster.kz`, чтобы разрешить всем
     с почтой этого домена.
8. Настройте **Session Duration** — как часто пользователь должен заново
   подтверждать email (например, раз в 24 часа).
9. Нажмите **Save** / **Create**.

После этого при заходе на `https://analysis.taxmaster.kz` пользователь
сначала увидит страницу Cloudflare с просьбой ввести email и код
подтверждения (придёт на почту), и только потом — обычный экран входа в
TaxMasterKZ. Доступ, не входящий в разрешённый список email, будет
автоматически заблокирован Cloudflare ещё до того, как запрос дойдёт до
вашего сервера.

## Если что-то пошло не так — куда смотреть

| Проблема | Что проверить |
|---|---|
| Сайт не открывается по IP | `docker compose ps` — контейнер запущен? `docker compose logs -f` — есть ошибки? Открыт ли порт (`sudo ufw status`)? |
| Сайт не открывается по домену | DNS настроен? (`ping analysis.taxmaster.kz`) `sudo systemctl status caddy` — Caddy запущен? |
| Нет HTTPS-замочка | Подождите пару минут после первого запроса — Caddy получает сертификат автоматически. Убедитесь, что порт 443 открыт. |
| После обновления пропали пользователи | Данные хранятся в `data/` — проверьте, что при обновлении (шаг 16) эта папка НЕ была перезаписана/удалена. |
| Забыли пароль администратора | См. README_DEPLOY.md, раздел 4 — можно создать нового администратора командой через `docker compose exec`. |
